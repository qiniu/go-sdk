# 通用输出格式契约（Single Source of Truth）

维护说明：更新时先改本文件，再同步到各 skill 的输出模板段落。各 skill 不得独立修改此骨架结构。

## 输出格式骨架

每个 skill 的输出必须包含以下结构（业务字段由各 skill 自行定义）：

```text
<skill 标题/结果概要>

<业务字段区>（由各 skill 定义）

结论: <一句话结论>
```

## 证据引用规范

- 每条结论或关键判断必须附带可追溯的证据来源。
- 证据类型及格式：
  - 命令输出：给出完整执行命令
  - 文件路径：`<file_path>:<line_number>`（如适用）
  - 链接：完整可点击 URL（GitHub issue/PR/构建页面等）
  - 日志摘要：关键行摘录（不超过 10 行），标注来源

## 边界约束

- 列表项超过 skill 约定上限时，用「共 N 项，详见 <位置>」概括。
- 禁止输出敏感信息（token、完整 AK/SK 等），仅可使用抽象描述。
- 结论必须为明确判定（如「已完成」「存在阻塞」「通过」「不通过」），不使用模糊表述。
