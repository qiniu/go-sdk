# 统一错误输出模板

各 skill 遇到错误时，按以下模板输出，确保用户能快速定位问题并提供所需信息。

## 1. 参数校验失败

当必填参数缺失或格式不合法时使用。

```text
参数校验失败

缺失/无效参数: <参数名>
要求: <参数格式说明>
收到: <实际值 或 "未提供">

用法: <skill名> <参数提示（来自 frontmatter argument-hint）>
示例: <一个合法的调用示例>
```

规则：
- 一次性列出所有校验失败项，不要逐个报告。
- 参数提示直接引用 frontmatter 的 `argument-hint`。
- 包含至少一个合法示例。

## 2. 步骤执行阻塞

当某步骤因权限、凭证、外部依赖等原因无法继续时使用。

```text
阻塞: 步骤 <N> - <步骤名称>

阻塞点: <具体描述>
已执行命令: <实际执行的命令>
报错信息: <关键错误行>
已尝试: <已做的排查/修复动作>

需要用户提供:
- <具体所需信息或操作 1>
- <具体所需信息或操作 2>
```

规则：
- 立即停止当前步骤，不进入下一步。
- 保留前序步骤的成功结果。
- "已执行命令"必须是实际执行的命令，不是模板。
- "报错信息"取关键行，不要完整日志。

## 3. 外部依赖缺失

当缺少环境变量、工具、网络连接等外部依赖时使用。

```text
外部依赖缺失: <依赖名称>

说明: <依赖的用途>
检查命令: <用于确认依赖的命令>
配置参考: <文档路径或设置说明>

当前影响: <跳过哪些步骤/使用什么降级方案>
```

规则：
- 明确说明跳过步骤的影响范围。
- 对于可降级的依赖，说明降级方案。

## 适用场景映射

| 错误类型 | 模板 | 典型场景 |
|----------|------|----------|
| 必填参数缺失 | 参数校验失败 | PR 编号未指定、URL 格式错误 |
| 参数值非法 | 参数校验失败 | issue 编号非正整数 |
| 权限不足 | 步骤执行阻塞 | gh 命令 403、API 无权限 |
| 命令执行失败 | 步骤执行阻塞 | make unittest 失败、git merge 冲突 |
| 环境变量缺失 | 外部依赖缺失 | GITHUB_TOKEN 未配置 |
| 工具未安装 | 外部依赖缺失 | staticcheck、gofumpt 不可用 |
